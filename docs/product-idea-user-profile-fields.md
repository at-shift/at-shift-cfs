# atshift User Profile Fields Product Idea

## 概要

WordPress のユーザープロフィール画面から不要な標準項目を外し、必要な入力欄だけを追加・整理するための小型プラグイン案。

atshift Fields の思想をユーザープロフィール画面に絞って展開する。投稿・固定ページ・カスタム投稿タイプ向けの atshift Fields 本体とは分け、ユーザー情報と `user_meta` の管理に特化する。

## 仮称

- atshift User Profile Fields
- atshift Profile Fields
- atshift User Fields

現時点では `atshift User Profile Fields` が最も説明的で分かりやすい。

## 価値

WordPress 標準のユーザープロフィール画面は、業務サイトや組織サイトでは不要な項目が多く、必要な項目を追加するにはコード実装や大型の会員管理プラグインが必要になりやすい。

このプラグインは、会員サイト全体を作るのではなく、管理画面のユーザープロフィールを実務用に整えることを目的にする。

主な価値:

- 不要な標準プロフィール項目を非表示にできる
- 必要なプロフィール項目を追加できる
- 入力内容を `user_meta` に保存できる
- ロールごとに表示・編集できる項目を制御できる
- 姓名、ふりがな、所属、部署、役職など、日本語の業務サイトに合う情報管理がしやすい
- atshift Fields 系の分かりやすいフィールド UI を流用できる

- 標準プロフィール項目の非表示
- ロール別の表示・編集制御
- 基本フィールド:

## 無料版と有料版の境界

採用方針:

- 無料版: ユーザーを分類する
- 有料版: 組織として管理する

無料版では、WordPress ユーザーを 1 つの分類で整理できる範囲に留める。部署、支部、チーム、講師区分、スタッフ区分など、単純なユーザー分類として使える状態を目指す。

有料版では、複数組織への所属、主所属・副所属、構成組織の階層管理、組織管理者による自組織ユーザー管理など、実務上の組織管理に必要な機能を扱う。

この境界により、無料版で便利さを体験してもらい、複雑な組織運用が必要になった場合に自然に有料版へ移行できる。

## 無料版候補: ユーザー分類

無料版には、ユーザー分類タクソノミーを入れる方針。

無料版で扱う範囲:

- ユーザー分類を 1 つだけ利用できる
- 階層あり
- ユーザープロフィール画面で分類を選択できる
- ユーザー一覧に分類列を表示できる
- 分類でユーザー一覧を絞り込める
- 分類別ユーザー一覧をショートコードで表示できる

想定用途:

- 部署別スタッフ
- 支部別メンバー
- 講師カテゴリ
- 店舗スタッフ分類
- 担当者区分

無料版の説明文案:

> ユーザーを部署・支部・チームなどの分類で整理できます。

## 有料版候補: 組織管理

有料版では、単なる分類ではなく組織運用を扱う。

有料版候補:

- 複数のユーザー分類
- 複数組織への所属
- 主所属 / 副所属
- 構成組織の階層管理
- 組織ごとのプロフィール項目
- 組織ごとの表示テンプレート
- 組織管理者が自組織ユーザーだけ編集
- 組織図表示
- 組織別メンバー一覧
- 登録時の組織自動付与

有料版への誘導文案:

> 無料版では、ユーザーを 1 つの分類で整理できます。複数組織への所属、主所属・副所属、組織管理者によるユーザー管理などが必要な場合は Pro 版をご利用ください。

## 組織管理との関係

将来的には、ユーザーを組織に紐付けることで以下のような運用が考えられる。

- 組織
  - 管理ユーザー
    - 組織に紐付いたユーザー
    - ユーザー登録権限あり
  - 組織ユーザー
    - 組織に紐付いたユーザー
    - ユーザー登録権限なし

組織に紐付いていないユーザーは候補や編集対象に出さない。表示名はログイン ID ではなく、姓名を基本にする。

ただし、組織という概念はサイトごとの差が大きいため、初期版では本体に組織管理を持たせない。将来的にフックや Pro 版で対応する。

## atshift Fields 本体との関係

atshift Fields 本体:

- 投稿
- 固定ページ
- カスタム投稿タイプ
- 投稿メタ
- 投稿編集画面の整理

atshift User Profile Fields:

- ユーザー
- ユーザープロフィール
- `user_meta`
- ユーザー編集画面の整理

本体に統合すると責務が広がりすぎるため、別プラグインにする方が分かりやすい。

ただし、UI、翻訳、ロール制御、フィールド定義、保存処理の考え方は atshift Fields から流用できる。

- ダウンロード URL 管理
- 返金・サポート方針

## WordPress.org への掲載方針

WordPress.org に公開する場合、無料で使える範囲を用意し、拡張機能や Pro 版に課金する形が基本になる。

ただし、このプラグインは小型の有料ツールとして始めたいので、初期段階では WordPress.org に出さない方針がよさそう。

代わりに、atshift Fields のドキュメントサイトや出力例ページに以下のような案内を置く。

例:

> ユーザープロフィール画面を業務用に整理したい場合は、atshift User Profile Fields も利用できます。

この形なら、atshift Fields の利用者に自然に紹介でき、WordPress.org の無料版運用ルールに縛られにくい。

## 実装ステップ案

1. プラグイン骨組みを作る
2. ユーザープロフィール画面に 1 つのテキストフィールドを表示する
3. `user_meta` に保存する
4. フィールドグループ風の管理 UI を作る
5. 複数フィールドに対応する
6. 標準プロフィール項目の非表示設定を追加する
7. ロール別の表示・編集制御を追加する
8. 出力用ショートコードまたは関数を追加する
9. 日本語 / 英語翻訳を整える
10. 販売・ライセンス配信の方法を決める
